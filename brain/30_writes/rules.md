# Writing Rules

Canonical rules for all posts under `brain/30_writes/`. Apply to every blog post and long-form piece.

---

## 1. Metadata & Filename

**Filename:** `YYYY-MM-DD-slug.md`  
Date first so posts sort chronologically. Use lowercase, hyphens for multi-word slugs.

**Frontmatter (YAML at top of file, before the first heading):**

```yaml
---
date: YYYY-MM-DD
tags:
  - tag-one
  - tag-two
author: Naren
contributors: []
status: draft   # or published
---
```

- **date:** Publication or draft date.
- **tags:** 5–10 relevant tags (concepts, domains, themes).
- **author:** Primary author.
- **contributors:** List of others who contributed; use `[]` if solo.
- **status:** `draft` or `published`.

---

## 2. Tone: Textbook / Informational

- **Serious, not casual.** No hype, no “hey reader,” no exclamation runs. Calm and precise.
- **Informational, not promotional.** Explain and analyze; don’t sell or oversell.
- **Precise language.** Name concepts clearly (e.g. “flywheel,” “self-reinforcing loop”). Avoid vague or buzzword-heavy phrasing.
- **Confident but not preachy.** State claims clearly; allow nuance. Use “A suggests B” or “This implies” where appropriate instead of “This proves.”
- **Historical and strategic depth when relevant.** When using history or case studies, use concrete detail (names, dates, mechanics) and clear cause–effect. Avoid superficial parallels.
- **No fluff.** Every paragraph should add information or argument. Cut filler.

---

## 3. Structure & Formatting

**Opening**
- One `#` title line.
- Optional subtitle on the next line in **bold**.
- `---` horizontal rule, then body.

**Sections**
- Use `##` for main sections.
- For long, multi-part pieces, use numbered section titles: `## I. Section Name`, `## II. Next Section`, etc.
- One idea per section. Section purpose should be clear from the heading.

**Between sections**
- Use `---` between major sections for visual break and scannability.

**Paragraphs**
- Short to medium length. Avoid single-sentence paragraphs unless for emphasis.
- One main idea per paragraph. Lead with the point, then support.

**Lists**
- Use `-` for unordered lists.
- Use `1. 2. 3.` for ordered sequences (processes, steps).
- After a list, add a one-line takeaway in *italics* when it reinforces the section.

**Emphasis**
- **Bold** for terms and key phrases when first introduced or when stressing.
- *Italics* for definitions, takeaways, and subtle emphasis (not for whole sentences).

**Conclusion**
- End with a short `## Conclusion` that restates the main thesis and one clear takeaway. No new arguments; synthesize.

---

## 4. Philosophy (from Weekly Writing)

- No hype. No fluff.
- Write to clarify thinking.
- Write to attract serious people.
